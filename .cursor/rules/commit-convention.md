# Git 커밋 컨벤션 (Google Style)

이 프로젝트의 커밋 메시지는 **Google 스타일** 형식을 따릅니다.  
커밋 작성·제안·리뷰 시 아래 규칙을 적용하세요.

## 형식 📝

```
<타입>(<범위>): <제목> [이모지]

<본문>

<푸터>
```

- **제목**: 필수. 한 줄로 요약. 이모지는 선택(권장).
- **본문**: 선택. 상세 설명이 필요할 때만.
- **푸터**: 선택. 이슈 번호, BREAKING CHANGE 등.

## 타입 (Type) & 이모지

| 타입       | 이모지 | 설명                                      | 예시                                            |
| ---------- | ------ | ----------------------------------------- | ----------------------------------------------- |
| `feat`     | ✨     | 새로운 기능 추가                          | `feat(courses): 코스 공개 기능 추가 ✨`         |
| `fix`      | 🐛     | 버그 수정                                 | `fix(ui): 모바일 모달 z-index 수정 🐛`          |
| `docs`     | 📚     | 문서 수정                                 | `docs(readme): 페이지 구조 설명 추가 📚`        |
| `style`    | 🎨     | 코드 포맷팅, 세미콜론 등 (기능 변경 없음) | `style(components): Prettier 적용 🎨`           |
| `refactor` | ♻️     | 리팩토링 (기능 변경 없음)                 | `refactor(services): 여행 서비스 로직 개선 ♻️`  |
| `perf`     | ⚡     | 성능 개선                                 | `perf(api): 코스 조회 쿼리 최적화 ⚡`           |
| `test`     | 🧪     | 테스트 추가/수정                          | `test(courses): 코스 생성 테스트 추가 🧪`       |
| `chore`    | 🔧     | 빌드, 패키지, 설정 등                     | `chore(deps): Next.js 16.1.6 업그레이드 🔧`     |
| `ci`       | 🔄     | CI/CD 설정 변경                           | `ci(github): GitHub Actions 워크플로우 추가 🔄` |
| `build`    | 📦     | 빌드/번들 변경                            | `build(webpack): 번들 크기 최적화 📦`           |
| `revert`   | ↩️     | 이전 커밋 되돌리기                        | `revert: "feat(courses): 코스 공개 기능" ↩️`    |

**이모지 사용**: 제목 끝에 타입별 이모지를 붙이면 로그에서 한눈에 구분하기 좋습니다. 생략해도 됩니다.

## 범위 (Scope) 🎯

변경이 영향을 미치는 영역. 선택이지만 가능하면 명시.

- 예: `api`, `components`, `services`, `ui`, `hooks`, `utils`, `types`, `config`, `docs`, `web`, `courses`, `expense`, `anniversary` 등

## 제목 (Subject) 규칙 ✏️

- **50자 이내**
- **명령형, 현재형** (과거형·진행형 지양)
- **첫 글자 소문자** (문장이 아닌 명령형)
- **마침표(.)로 끝나지 않음**
- **무엇을 했는지 명확하게**

**좋은 예 ✅**

```
feat(courses): 코스 공개 기능 추가 ✨
fix(ui): 모바일에서 모달이 깨지는 문제 수정 🐛
refactor(api): Supabase 클라이언트 생성 로직 개선 ♻️
```

**나쁜 예 ❌**

```
feat: 코스 공개 기능을 추가했습니다. (#12)   ← 제목이 설명문, 길음
fix: 모달 버그 수정                          ← 범위·내용 불명확
refactor: 코드 개선                           ← 구체적이지 않음
```

## 본문 (Body) 📄

- 제목과 본문 사이 **빈 줄 하나**
- 72자마다 줄바꿈
- 무엇을·왜 변경했는지, 필요 시 어떻게 변경했는지 작성

## 푸터 (Footer) 🔗

- `Closes #이슈번호` — 이슈를 닫음
- `Fixes #이슈번호` — 버그 이슈 수정
- `Refs #이슈번호` — 관련 이슈 참조
- `BREAKING CHANGE:` — 호환 깨짐이 있을 때 설명

## 체크리스트 (커밋 전) ✅

- [ ] 타입이 올바른가?
- [ ] 범위가 명확한가?
- [ ] 제목이 50자 이내이고 명령형인가?
- [ ] (선택) 타입에 맞는 이모지를 붙였는가?
- [ ] 본문이 필요한 경우 충분한가?
- [ ] 관련 이슈 번호가 있으면 푸터에 넣었는가?
- [ ] 브레이킹 체인지는 문서화했는가?

## 참고 💡

- 하나의 커밋은 **하나의 논리적 변경**만 포함.
- 팀원이 이해할 수 있도록 **명확하고 일관된** 메시지 유지.
- 이모지를 쓰면 `git log`에서 타입을 빠르게 구분할 수 있습니다.
